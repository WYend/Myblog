### **一、项目技术栈 (Tech Stack)**

- **后端**: Spring Boot 3
- **数据层**: MyBatis-Plus
- **前端**: Vue 3 + Element Plus
- **构建工具**: 大概率是 Vite

---

### **二、项目角色与核心功能 (Roles & Core Features)**

#### 1. **用户角色**

- **访客（未登录用户）**: 浏览首页、文章列表、文章详情。
- **管理员（我本人）**: 登录后台，拥有所有数据的 CRUD 权限。

#### 2. **核心功能模块（优先级排序）**

- **高优先级**: 首页文章列表、文章详情页（Markdown 渲染）。
- **中优先级**: 分类/标签筛选功能。

---

### **三、前端架构现状 (Current Frontend Structure)**

#### 1. **已完成部分**

- **路由配置** (`router/index.js`): 已配置首页、文章列表、详情、分类/标签路由。
- **基础布局组件**:
  - `Header.vue`: 顶部导航栏，含搜索框和暗色模式切换。
  - `Footer.vue`: 页脚信息。
  - `Sidebar.vue`: 侧边栏，含关于我、分类、标签云（目前是模拟数据）。
  - `Layout.vue`: 整体布局结构，整合了以上组件。
- **视图页面占位**:
  - `Home.vue`, `ArticleList.vue`, `ArticleDetail.vue`, `CategoryTag.vue` (已有基本路由，内容待实现)。
- **身份验证页面**:
  - `Login.vue`: 登录页面，包含表单验证和模拟登录逻辑。
  - `Register.vue`: 注册页面，包含表单验证和模拟注册逻辑。
- **全局样式**: 已在 `App.vue` 中设置了基础样式和暗色模式支持。

#### 2. **待实现部分**

- **首页内容**: 轮播图、最新文章展示、热门分类/标签云。
- **文章卡片组件** (`ArticleCard.vue`): 用于在列表中展示文章摘要。
- **文章列表页**: 集成筛选、排序、分页功能，并与后端 API 对接。
- **文章详情页**: Markdown 渲染、文章导航、相关文章推荐。
- **分类/标签页**: 根据参数展示对应文章列表。
- **后台管理界面**: 完整的文章、分类、标签的 CRUD 操作界面（尚未开始）。
- **API 集成**: 所有前端页面需要与后端 Spring Boot API 进行连接（目前是模拟数据）。
- **状态管理**: 考虑使用 Pinia 来管理用户登录状态、文章数据等（尚未引入）。

---

### **四、设计规范 (Design Specifications)**

- **配色方案**: 主色 `#409EFF` (Element Plus 蓝)，辅以绿、黄等标准色。
- **布局**: 响应式设计，大屏（侧栏+主内容），小屏（单栏布局，汉堡菜单）。
- **风格**: 简洁现代，突出内容，使用 Element Plus 组件保持一致性。
